## 快速开始

### 1. 准备环境

- JDK 1.8
- Maven 3.6+
- Node.js 18+
- MySQL 5.7 / 8.0
- Redis

### 2. 初始化数据库

建议按以下顺序导入：

1. `canglian/sql/canglian_20250522.sql`
2. `canglian/sql/ec_core_20260125.sql`
3. `canglian/sql/quartz.sql`
4. `canglian/sql/sample_data.sql`

### 3. 启动后端

```bash
cd canglian
mvn package -DskipTests
```

启动类：

```text
com.canglian.CanglianApplication
```

### 4. 启动前端

```bash
cd canglian-vue3
npm install
npm run dev
```

### 5. 默认访问

- 后端接口：`http://localhost:7565`
- 前端开发地址：以 `npm run dev` 实际输出端口为准

## 项目简介

仓链管理系统是一套基于 `RuoYi` / `RuoYi-Vue` 体系进行二次开发的前后端分离管理系统。

项目在保留 RuoYi 原有权限管理、系统管理、监控日志、定时任务、代码生成等通用能力的基础上，结合实际业务场景扩展了仓储管理、库存流转、基础资料管理和财务核算等模块，用于支撑企业内部的业务协同与经营管理。

当前系统的主要定位是：

- 以 RuoYi 作为底层开发框架
- 以仓储业务为核心业务主线
- 以财务数据为辅助管理主线
- 以权限、日志、监控、代码生成作为基础平台能力
- 以金蝶云星辰商贸标准版/专业版的轻量替代为阶段性对标方向

## 二次开发说明

本项目属于基于 RuoYi 的二次开发项目，主要体现为以下几个方面：

- 保留了 RuoYi 多模块后端结构
- 保留了 RuoYi 的系统管理与权限控制体系
- 保留了 RuoYi 的监控、日志、定时任务、代码生成等基础能力
- 在此基础上新增并扩展了仓储、库存、客户、供应商、产品、财务等业务模块
- 对系统标识、项目名称、模块命名进行了业务化调整，形成当前的 `canglian` 项目结构

如果从项目来源理解，这个系统不是从零开始独立搭建，而是在 RuoYi 框架能力之上进行业务落地和功能扩展形成的业务系统。

## 部署流程

建议按以下顺序完成项目部署：

### 1. 准备基础环境

- 安装并配置 JDK
- 安装并配置 Maven
- 安装并配置 Node.js
- 安装并启动 MySQL
- 安装并启动 Redis

- 创建 `canglian` 数据库
- 导入主业务脚本
- 按实际需要导入 Quartz 相关脚本
- 按实际需要导入示例数据脚本

建议导入顺序：

1. `canglian_20250522.sql`
2. `ec_core_20260125.sql`
3. `quartz.sql`
4. `sample_data.sql`

### 3. 修改后端配置

重点检查以下配置是否正确：

- MySQL 连接地址
- MySQL 用户名和密码
- Redis 连接地址
- Redis 密码
- 上传目录配置
- 服务端口配置

### 4. 启动后端服务

- 编译打包后端工程
- 启动 `canglian-admin` 模块
- 确认接口服务正常访问

### 5. 启动前端服务

- 安装前端依赖
- 启动本地开发服务
- 确认前端可正常访问后端接口

### 6. 构建生产包

- 后端执行 Maven 打包
- 前端执行生产构建
- 将前后端产物部署到目标环境

## 部署检查清单

正式部署前，建议逐项检查以下内容：

- 是否已创建 `canglian` 数据库并完成全部脚本导入
- 是否已根据部署环境修改数据库连接地址、账号与密码
- 是否已根据部署环境修改 Redis 地址、端口与认证信息
- 是否已确认文件上传目录 `profile` 指向服务器可写路径
- 是否已确认后端端口 `7565` 未被其他服务占用
- 是否已完成前端 `npm install` 与 `npm run build:prod` 构建
- 是否已确认反向代理配置与 `/prod-api` 接口转发路径一致
- 是否已检查默认账号密码并在上线前完成修改
- 是否已确认日志目录、附件目录、缓存目录具备读写权限

## 生产部署建议

为了保证系统稳定运行，建议在生产环境中按以下方式处理：

- 数据库、Redis、上传目录不要继续使用开发默认配置
- 前端建议通过 Nginx 托管静态资源，后端单独以 jar 方式运行
- 上传目录建议使用独立磁盘路径，例如 `d:/canglian/uploadPath` 或 Linux 下独立目录
- 建议对数据库执行定期备份，对上传文件执行增量备份
- 建议通过 Nginx 或网关统一做 HTTPS、跨域与访问控制
- 建议对定时任务、财务数据操作日志、登录日志进行定期巡检
- 若系统继续沿用 RuoYi 代码生成能力，新增模块时应保持现有分层规范与权限模型一致

### 1. 初始化数据库

根据 `canglian/sql` 目录中的脚本初始化数据库，并确认数据库名称、账号密码与配置文件保持一致。

- `canglian-admin`
