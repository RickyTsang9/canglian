# 仓链管理系统

## 项目简介

仓链管理系统是一套基于 `RuoYi` / `RuoYi-Vue` 体系进行二次开发的前后端分离管理系统。

项目在保留 RuoYi 原有权限管理、系统管理、监控日志、定时任务、代码生成等通用能力的基础上，结合实际业务场景扩展了仓储管理、库存流转、基础资料管理和财务核算等模块，用于支撑企业内部的业务协同与经营管理。

当前系统的主要定位是：

- 以 RuoYi 作为底层开发框架
- 以仓储业务为核心业务主线
- 以财务数据为辅助管理主线
- 以权限、日志、监控、代码生成作为基础平台能力

## 二次开发说明

本项目属于基于 RuoYi 的二次开发项目，主要体现为以下几个方面：

- 保留了 RuoYi 多模块后端结构
- 保留了 RuoYi 的系统管理与权限控制体系
- 保留了 RuoYi 的监控、日志、定时任务、代码生成等基础能力
- 在此基础上新增并扩展了仓储、库存、客户、供应商、产品、财务等业务模块
- 对系统标识、项目名称、模块命名进行了业务化调整，形成当前的 `canglian` 项目结构

如果从项目来源理解，这个系统不是从零开始独立搭建，而是在 RuoYi 框架能力之上进行业务落地和功能扩展形成的业务系统。

## 模块说明

### 后端模块

- `canglian-admin`：系统启动入口，负责接口暴露、权限接入和后台管理主流程
- `canglian-common`：通用工具、基础封装、统一返回、公共配置
- `canglian-framework`：安全认证、拦截器、权限处理、系统基础框架能力
- `canglian-system`：系统管理与业务管理核心模块
- `canglian-quartz`：定时任务与调度相关能力
- `canglian-generator`：代码生成模块，保留了 RuoYi 常用开发能力

### 业务模块概览

当前业务代码主要集中在以下方向：

- 主数据模块：客户、供应商、产品、仓库、库位
- 仓储模块：入库、出库、库存、库存流水、调拨、盘点、退货
- 财务模块：应收、应付、收款、付款、费用、账户、核销、坏账、报表
- 平台模块：用户、角色、菜单、日志、监控、任务、代码生成

## 部署流程

建议按以下顺序完成项目部署：

### 1. 准备基础环境

- 安装并配置 JDK
- 安装并配置 Maven
- 安装并配置 Node.js
- 安装并启动 MySQL
- 安装并启动 Redis

### 2. 初始化数据库

- 创建 `canglian` 数据库
- 导入主业务脚本
- 按实际需要导入 Quartz 相关脚本
- 按实际需要导入示例数据脚本

建议导入顺序：

1. `canglian_20250522.sql`
2. `ec_core_20260125.sql`
3. `quartz.sql`
4. `sample_data.sql`

### 3. 修改后端配置

重点检查以下配置是否正确：

- MySQL 连接地址
- MySQL 用户名和密码
- Redis 连接地址
- Redis 密码
- 上传目录配置
- 服务端口配置

### 4. 启动后端服务

- 编译打包后端工程
- 启动 `canglian-admin` 模块
- 确认接口服务正常访问

### 5. 启动前端服务

- 安装前端依赖
- 启动本地开发服务
- 确认前端可正常访问后端接口

### 6. 构建生产包

- 后端执行 Maven 打包
- 前端执行生产构建
- 将前后端产物部署到目标环境

## 后端启动方式

### 1. 初始化数据库

根据 `canglian/sql` 目录中的脚本初始化数据库，并确认数据库名称、账号密码与配置文件保持一致。

### 2. 修改数据源配置

根据本地环境调整以下配置：

- MySQL 地址、端口、数据库名
- 用户名与密码
- Redis 地址、端口、密码
- 文件上传目录

### 3. 启动后端

在 `canglian` 目录执行：

```bash
mvn package -DskipTests
```

或直接运行启动类：

```text
com.canglian.CanglianApplication
```

后端主启动模块为：

- `canglian-admin`

后端打包完成后，默认产物位于：

- `canglian/canglian-admin/target/`

## 前端启动方式

在 `canglian-vue3` 目录执行：

```bash
npm install
npm run dev
```

前端生产构建命令：

```bash
npm run build:prod
```

前端构建完成后，默认产物位于：

- `canglian-vue3/dist/`

## 初始化建议

首次接手该项目时，建议优先完成以下检查：

- 确认数据库脚本是否与当前环境一致
- 确认 Redis 是否已启动并可连接
- 确认前后端接口代理路径是否匹配
- 确认上传目录在服务器上具备读写权限
- 确认系统账号、角色、菜单数据是否已初始化
- 确认定时任务表是否已创建
